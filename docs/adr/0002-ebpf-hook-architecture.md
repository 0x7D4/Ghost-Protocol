# ADR-0002: eBPF Hook Architecture — XDP vs TC

## Status

Accepted

## Date

2026-04-18

## Context

Ghost-Protocol implements Moving Target Defense (MTD) involving OS persona morphing, IP allowlisting, scanner detection, and port knocking sequences. We need to decide the proper split of responsibilities between the XDP (eXpress Data Path) ingress hook and the TC (Traffic Control) egress hook.

Earlier designs incorrectly assumed XDP would rewrite inbound SYN packets to normalize them. However, modifying inbound packets alters the traffic as seen by the local host and does not affect what an external scanner (like Nmap) perceives.

## Decision

The packet inspection and modification pipeline is split strictly into:

1. **XDP Ingress Hook (Read-Only)**
   - Operates strictly in a read-only capacity.
   - Detects scanner patterns and incoming port knocks.
   - Maintains and updates BPF map state (e.g., counting unique ports per `src_ip` for scanner detection).
   - Validates incoming packets against the `ALLOWLIST` map.
   - **Does not modify packets**; allows them to pass untouched to the network stack.

2. **TC Egress Hook (Mutating)**
   - Intercepts **outbound SYN-ACK** responses generated by the local network stack.
   - Modifies the packet to spoof the active OS persona (manipulating TTL, TCP window size, TCP options like SACK/WScale, and TCP timestamps, as well as IP ID sequences).
   - Checks the `ALLOWLIST` and `CONNTRACK` maps to optionally bypass morphing for established trusted sessions.
   - Recalculates necessary IPv4 and TCP checksums upon modification.

## Rationale

- **Accuracy for OS Fingerprinting**: External scanners fingerprint a host based on its *responses*, not how it handles inbound traffic. Modifying the outbound SYN-ACK precisely covers the features Nmap and other tools map to particular operating systems.
- **Complexity and Performance**: XDP is incredibly fast and drops bad packets early. Keeping XDP read-only simplifies the verifier's job and keeps state updates lightweight. TC is better suited for packet expansion and deeper modifications on standard outbound paths.
- **State Management**: A BPF `LRU_HASH` map (Connection Tracker) ensures that only the initial SYN-ACK is morphed or recorded, leaving subsequent traffic untouched to reduce overhead. Allowlisted IP addresses bypass both hooks quickly via a `HASH` map lookup.

## Consequences

- **Positive**: Correctly spoon-feeds fake signatures to external OS detection tools. Keeps XDP extremely lean and performant.
- **Negative**: Requires correctly recalculating checksums on outbound paths in TC, adding slight CPU overhead to legitimate outbound SYN-ACK packets.
